# core_pipe Design Spec

## 1. Scope

`core_pipe` is the initial pipeline-control integration skeleton for the
planned simple in-order core. This milestone intentionally does not execute
instruction semantics; it establishes the pipeline state machinery that later
connects decode, regfile, ALU, LSU, CSR, trap, hazard, and writeback logic.


## 3. Design

`core_pipe` does not own a fetch PC register. A normal accepted frontend stream
beat captures `{instr_pc_i, instr_i, instr_fault_i}` into IF/ID. The frontend is
responsible for boot PC initialization, sequential PC increment, and redirect PC
loading.

The frontend stream beat is accepted when:

```text
!fetch_stall_i && !decode_stall_i && !redirect_valid_i
```

Redirect has highest priority, clears both pipeline slots, deasserts
`instr_ready_o`, and forwards the target on `redirect_valid_o/redirect_pc_o`.

If `execute_bubble_i` is asserted, the EX/WB slot is cleared. Otherwise, when
decode is not stalled, IF/ID advances into EX/WB.

IF/ID captures an instruction stream beat in the same cycle it advances. If no
stream beat is accepted and decode is not stalled, IF/ID is cleared.

## 4. Pipeline State Diagram

![core_pipe pipeline state](images/core_pipe_state.png)

PNG generated by `docs/tools/render_state_pngs.py`.

```text
Reset:
  IF/ID.valid = 0, IF/ID.pc = 0, IF/ID.instr = NOP, IF/ID.fault = 0
  EX/WB.valid = 0, EX/WB.pc = 0, EX/WB.instr = NOP, EX/WB.fault = 0

Clock-edge priority:

  redirect_valid_i
        |
        v
  redirect_valid_o = 1, redirect_pc_o = redirect_pc_i
  IF/ID <- invalid NOP
  EX/WB <- invalid NOP

  else normal pipeline update:

    instr_fire = instr_valid_i &&
                 instr_ready_o

    if execute_bubble_i:
      EX/WB <- invalid NOP
    else if !decode_stall_i:
      EX/WB <- old IF/ID
    else:
      EX/WB holds old value

    if instr_fire:
      IF/ID <- {valid=1, pc=instr_pc_i, instr=instr_i,
                fault=instr_fault_i}
    else if !decode_stall_i:
      IF/ID <- invalid NOP
    else:
      IF/ID holds old value
```

`redirect_valid_i` therefore beats instruction acceptance, decode advance, and
execute bubble. A load-use sequence is represented by asserting
`decode_stall_i` and `execute_bubble_i` together: IF/ID holds while EX/WB is
cleared.

## 5. Target Support

The module is target-neutral synthesizable sequential logic. No IC or
Virtex-7-specific primitive is required.
